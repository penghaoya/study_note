# Drizzle-Orm

## 运算符

### 比较运算符

#### eq:等于

- 使用场景: 用于判断字段值是否等于某个值，最常用于 WHERE 查询条件。
- 注意事项: 比较字符串时不要忘记加引号，eq 返回的是 SQL 表达式，不是布尔值。

```typescript
db.select().from(users).where(eq(users.name, 'Tom'))
```

#### ne:不等于

- 使用场景: 判断字段值不等于指定值。
- 注意事项: 不等于条件可能无法使用索引，注意性能问题。

```typescript
db.select().from(users).where(ne(users.age, 18))
```

#### gt:大于

- 使用场景: 常用于数字或时间类型的比较。
- 注意事项: 仅适用于可比较类型（如整数、浮点数、日期等）。

```typescript
db.select().from(users).where(gt(users.score, 80))
```

#### gte:大于等于

- 使用场景: 与 gt 类似，但包含等于。

```typescript
db.select().from(users).where(gte(users.age, 18))
```

#### lt:小于

- 使用场景: 小于比较，常见于分页、时间区间等场景。

```typescript
db.select().from(users).where(lt(users.price, 100))
```

#### lte:小于等于

```typescript
db.select().from(users).where(lte(users.createdAt, '2024-01-01'))
```

### 逻辑运算符

#### and:且

- 使用场景: 组合多个条件，所有条件都为真时匹配。
- 注意事项: 可以嵌套使用，不要传入 undefined 条件，否则报错。

```typescript
db.select()
  .from(users)
  .where(and(eq(users.role, 'admin'), gt(users.age, 30)))
```

#### or:或

- 使用场景: 只要满足任一条件即匹配。

```typescript
db.select()
  .from(users)
  .where(or(eq(users.city, 'Beijing'), eq(users.city, 'Shanghai')))
```

#### not:取反

- 使用场景: 对条件结果取反。

```typescript
db.select()
  .from(users)
  .where(not(eq(users.isActive, true)))
```

### 集合与匹配运算符

#### inArray

- 使用场景: 字段值在指定数组中。
- 注意事项: 传入的数组不能为空，否则 SQL 报错，适用于常量数组过滤。

```typescript
db.select()
  .from(users)
  .where(inArray(users.id, [1, 2, 3]))
```

#### notInArray

- 使用场景: 字段值不在指定数组中。

```typescript
db.select()
  .from(users)
  .where(notInArray(users.status, ['deleted', 'banned']))
```

#### isNull / isNotNull

- 使用场景: 判断字段是否为 NULL 或非空。
- 注意事项: 不能用 eq(field, null) 判断。

```typescript
db.select().from(users).where(isNull(users.deletedAt))
```

#### exists / notExists

- 使用场景: 用于子查询条件，判断子查询是否有结果。

```typescript
const subQuery = db.select({ id: orders.userId }).from(orders)
db.select().from(users).where(exists(subQuery))
```

### 字符串运算符

#### like

- 使用场景: 模糊匹配字符串，支持 % 通配符。
- 注意事项: % 表示任意多个字符，\_ 表示单个字符。

```typescript
db.select().from(users).where(like(users.name, '%Tom%'))
```

#### ilike

- 使用场景: 与 like 类似，但不区分大小写（部分数据库如 SQLite 不支持）。

```typescript
db.select().from(users).where(ilike(users.email, '%gmail.com'))
```

### 范围运算符

#### between / notBetween

- 使用场景: 判断字段值是否在某个范围内。
- 注意事项: 两个参数必须是 [min, max]，可用于数字、时间等类型。

```typescript
db.select().from(users).where(between(users.age, 18, 30))
```

### 算术运算符

#### add / sub / mul / div

- 使用场景: 用于计算字段值，如统计、计算总价。
- 注意事项: 多用于 SELECT 字段计算，不常在 WHERE 使用。

```typescript
import { add, mul } from 'drizzle-orm'
db.select({
  total: add(users.balance, 100),
  priceWithTax: mul(products.price, 1.1)
})
```

### 自定义 SQL 表达式

#### sql

- 使用场景: 当 Drizzle 无法直接表达某种条件时，可以用原生 SQL。
- 注意事项: 注意防止 SQL 注入（不要拼接用户输入），建议仅在复杂逻辑或性能优化时使用。

```typescript
import { sql } from 'drizzle-orm'
db.select()
  .from(users)
  .where(sql`LENGTH(${users.name}) > 3`)
```

### 其他常用表达式

#### exists / any / all

- 使用场景: 多用于子查询或聚合条件。

```typescript
db.select()
  .from(users)
  .where(sql`${users.id} = ANY(${sql`SELECT user_id FROM orders`})`)
```

### 总结

| 类型   | 常用函数                 | 场景     | 注意                          |
| ------ | ------------------------ | -------- | ----------------------------- |
| 比较   | eq, ne, gt, gte, lt, lte | 条件筛选 | 忘记使用 `sql` 封装原始表达式 |
| 逻辑   | and, or, not             | 组合条件 | 传入 undefined 条件会报错     |
| 集合   | inArray, notInArray      | 多选过滤 | 数组不能为空                  |
| 字符串 | like, ilike              | 模糊匹配 | `%` 写法错误                  |
| 范围   | between                  | 区间过滤 | 参数顺序错误                  |
| 算术   | add, sub, mul, div       | 计算字段 | 不支持非数值类型              |
| 其他   | sql, exists              | 高级用法 | 易写出不安全 SQL              |
