# Drizzle-ORM 学习笔记

## 基本配置

### 安装依赖

```bash
npm install drizzle-orm
npm install -D drizzle-kit
# 数据库驱动（选择一个）
npm install pg @types/pg          # PostgreSQL
npm install mysql2                # MySQL
npm install better-sqlite3         # SQLite
```

### 数据库连接配置

```typescript
// PostgreSQL
import { drizzle } from 'drizzle-orm/node-postgres'
import { Pool } from 'pg'

const pool = new Pool({
  connectionString: process.env.DATABASE_URL
})
const db = drizzle(pool)

// MySQL
import { drizzle } from 'drizzle-orm/mysql2'
import mysql from 'mysql2/promise'

const connection = await mysql.createConnection({
  host: 'localhost',
  user: 'root',
  database: 'test'
})
const db = drizzle(connection)

// SQLite
import { drizzle } from 'drizzle-orm/better-sqlite3'
import Database from 'better-sqlite3'

const sqlite = new Database('sqlite.db')
const db = drizzle(sqlite)
```

## 模型定义

### 基本表定义

```typescript
import {
  pgTable,
  serial,
  text,
  integer,
  timestamp,
  boolean
} from 'drizzle-orm/pg-core'

export const users = pgTable('users', {
  id: serial('id').primaryKey(),
  name: text('name').notNull(),
  email: text('email').notNull().unique(),
  age: integer('age'),
  isActive: boolean('is_active').default(true),
  createdAt: timestamp('created_at').defaultNow(),
  updatedAt: timestamp('updated_at').defaultNow()
})

export const posts = pgTable('posts', {
  id: serial('id').primaryKey(),
  title: text('title').notNull(),
  content: text('content'),
  authorId: integer('author_id').references(() => users.id),
  createdAt: timestamp('created_at').defaultNow()
})
```

### 关系定义

```typescript
import { relations } from 'drizzle-orm'

export const usersRelations = relations(users, ({ many }) => ({
  posts: many(posts)
}))

export const postsRelations = relations(posts, ({ one }) => ({
  author: one(users, {
    fields: [posts.authorId],
    references: [users.id]
  })
}))
```

## 增删改查

### 查询 (Select)

```typescript
// 基本查询
const allUsers = await db.select().from(users)

// 选择特定字段
const userNames = await db
  .select({
    id: users.id,
    name: users.name
  })
  .from(users)

// 条件查询
const activeUsers = await db
  .select()
  .from(users)
  .where(eq(users.isActive, true))

// 关联查询
const usersWithPosts = await db
  .select()
  .from(users)
  .leftJoin(posts, eq(users.id, posts.authorId))

// 使用关系查询
const usersWithPosts = await db.query.users.findMany({
  with: {
    posts: true
  }
})

// 分页
const paginatedUsers = await db.select().from(users).limit(10).offset(20)

// 排序
const sortedUsers = await db
  .select()
  .from(users)
  .orderBy(users.createdAt, users.name)

// 聚合查询
const userCount = await db
  .select({
    count: count()
  })
  .from(users)
```

### 插入 (Insert)

```typescript
// 插入单条记录
const newUser = await db
  .insert(users)
  .values({
    name: 'John Doe',
    email: 'john@example.com',
    age: 30
  })
  .returning()

// 插入多条记录
const newUsers = await db
  .insert(users)
  .values([
    { name: 'Alice', email: 'alice@example.com' },
    { name: 'Bob', email: 'bob@example.com' }
  ])
  .returning()

// 冲突处理 (PostgreSQL)
const upsertUser = await db
  .insert(users)
  .values({ name: 'John', email: 'john@example.com' })
  .onConflictDoUpdate({
    target: users.email,
    set: { name: 'John Updated' }
  })
```

### 更新 (Update)

```typescript
// 更新记录
const updatedUser = await db
  .update(users)
  .set({
    name: 'John Smith',
    updatedAt: new Date()
  })
  .where(eq(users.id, 1))
  .returning()

// 批量更新
await db
  .update(users)
  .set({ isActive: false })
  .where(lt(users.createdAt, '2023-01-01'))
```

### 删除 (Delete)

```typescript
// 删除记录
const deletedUser = await db.delete(users).where(eq(users.id, 1)).returning()

// 批量删除
await db.delete(users).where(eq(users.isActive, false))
```

## 运算符

### 快速参考

| 类型   | 函数                       | 用途     | 注意事项               |
| ------ | -------------------------- | -------- | ---------------------- |
| 比较   | `eq, ne, gt, gte, lt, lte` | 条件筛选 | 字符串加引号           |
| 逻辑   | `and, or, not`             | 组合条件 | 避免 undefined         |
| 集合   | `inArray, notInArray`      | 多选过滤 | 数组不能为空           |
| 空值   | `isNull, isNotNull`        | 空值判断 | 不能用 eq(field, null) |
| 字符串 | `like, ilike`              | 模糊匹配 | % 和 \_ 通配符         |
| 范围   | `between, notBetween`      | 区间过滤 | 参数顺序 min, max      |
| 算术   | `add, sub, mul, div`       | 字段计算 | 仅用于 SELECT          |
| 自定义 | `sql`                      | 原生 SQL | 防止注入               |

### 比较运算符

```typescript
// eq: 等于 - 最常用于 WHERE 查询条件
// 注意: 比较字符串时不要忘记加引号，eq 返回的是 SQL 表达式，不是布尔值
db.select().from(users).where(eq(users.name, 'Tom'))

// ne: 不等于 - 判断字段值不等于指定值
// 注意: 不等于条件可能无法使用索引，注意性能问题
db.select().from(users).where(ne(users.age, 18))

// gt: 大于 - 常用于数字或时间类型的比较
// 注意: 仅适用于可比较类型（如整数、浮点数、日期等）
db.select().from(users).where(gt(users.score, 80))

// gte: 大于等于 - 与 gt 类似，但包含等于
db.select().from(users).where(gte(users.age, 18))

// lt: 小于 - 常见于分页、时间区间等场景
db.select().from(users).where(lt(users.price, 100))

// lte: 小于等于
db.select().from(users).where(lte(users.createdAt, '2024-01-01'))
```

### 逻辑运算符

```typescript
// and: 且 - 组合多个条件，所有条件都为真时匹配
// 注意: 可以嵌套使用，不要传入 undefined 条件，否则报错
db.select()
  .from(users)
  .where(and(eq(users.role, 'admin'), gt(users.age, 30)))

// or: 或 - 只要满足任一条件即匹配
db.select()
  .from(users)
  .where(or(eq(users.city, 'Beijing'), eq(users.city, 'Shanghai')))

// not: 取反 - 对条件结果取反
db.select()
  .from(users)
  .where(not(eq(users.isActive, true)))
```

### 集合与匹配运算符

```typescript
// inArray: 字段值在指定数组中
// 注意: 传入的数组不能为空，否则 SQL 报错，适用于常量数组过滤
db.select()
  .from(users)
  .where(inArray(users.id, [1, 2, 3]))

// notInArray: 字段值不在指定数组中
db.select()
  .from(users)
  .where(notInArray(users.status, ['deleted', 'banned']))

// isNull/isNotNull: 判断字段是否为 NULL 或非空
// 注意: 不能用 eq(field, null) 判断
db.select().from(users).where(isNull(users.deletedAt))
db.select().from(users).where(isNotNull(users.email))

// exists/notExists: 用于子查询条件，判断子查询是否有结果
const subQuery = db.select({ id: orders.userId }).from(orders)
db.select().from(users).where(exists(subQuery))
```

### 字符串运算符

```typescript
// like: 模糊匹配字符串，支持 % 通配符
// 注意: % 表示任意多个字符，_ 表示单个字符
db.select().from(users).where(like(users.name, '%Tom%'))

// ilike: 与 like 类似，但不区分大小写
// 注意: 部分数据库如 SQLite 不支持
db.select().from(users).where(ilike(users.email, '%gmail.com'))
```

### 范围运算符

```typescript
// between/notBetween: 判断字段值是否在某个范围内
// 注意: 两个参数必须是 [min, max]，可用于数字、时间等类型
db.select().from(users).where(between(users.age, 18, 30))
db.select().from(users).where(notBetween(users.score, 0, 60))
```

### 算术运算符

```typescript
// add/sub/mul/div: 用于计算字段值，如统计、计算总价
// 注意: 多用于 SELECT 字段计算，不常在 WHERE 使用
import { add, sub, mul, div } from 'drizzle-orm'

db.select({
  total: add(users.balance, 100),
  priceWithTax: mul(products.price, 1.1),
  discount: sub(products.price, 10),
  average: div(users.totalScore, users.testCount)
})
```

### 自定义 SQL

```typescript
import { sql } from 'drizzle-orm'

// sql: 当 Drizzle 无法直接表达某种条件时，可以用原生 SQL
// 注意: 注意防止 SQL 注入（不要拼接用户输入），建议仅在复杂逻辑或性能优化时使用
db.select()
  .from(users)
  .where(sql`LENGTH(${users.name}) > 3`)

// exists/any/all: 多用于子查询或聚合条件
db.select()
  .from(users)
  .where(sql`${users.id} = ANY(${sql`SELECT user_id FROM orders`})`)
```

## 事务

### 基本事务

```typescript
// 简单事务
await db.transaction(async tx => {
  await tx.insert(users).values({ name: 'John', email: 'john@example.com' })
  await tx.insert(posts).values({ title: 'Hello World', authorId: 1 })
})

// 事务回滚
try {
  await db.transaction(async tx => {
    await tx.insert(users).values({ name: 'John', email: 'john@example.com' })
    throw new Error('Something went wrong')
    await tx.insert(posts).values({ title: 'Hello World', authorId: 1 })
  })
} catch (error) {
  console.log('Transaction rolled back')
}
```

### 事务隔离级别

```typescript
// 设置隔离级别
await db.transaction(
  async tx => {
    // 事务逻辑
  },
  {
    isolationLevel: 'read committed' // 'read uncommitted' | 'read committed' | 'repeatable read' | 'serializable'
  }
)
```

## 查询构建器高级用法

### 子查询

```typescript
// 子查询
const subquery = db
  .select({
    authorId: posts.authorId,
    postCount: count(posts.id).as('post_count')
  })
  .from(posts)
  .groupBy(posts.authorId)
  .as('post_counts')

const usersWithPostCount = await db
  .select({
    id: users.id,
    name: users.name,
    postCount: subquery.postCount
  })
  .from(users)
  .leftJoin(subquery, eq(users.id, subquery.authorId))
```
